Getting started

    install

  • ```console $ pip install pudb ```

    • run a script through pudb

  • ```console $ python -m pudb.run script.py ```

    • run tests with pudb

  • ```console $ nosetests --pudb ```

    • within your code

  • ```python from pudb import set_trace set_trace() ```

pudb cheat sheet

  • b - breakpoint
  • s - step into
  • n - next statement
  • c - continue
  • t - teleport (execute until)
  • / - search
  • m - view a module's source code
  • q - quit

common debugger patterns

  • handle errors over long iterations
  • access local variables with names like: 'n', 'c', 'q', 's'
  • profile: memory, counts, CPU
  • flake8 (linter)

handle errors over long iterations

```python for u in users: u.set_home_location() ```

reduce your footprint

  • important for long running processes: think streams
  • watch your memory (processes and CPU too): htop and memory_profiler
  • watch the network: use a network analyzer or log network requests

```python sync_to_other_db( session, map( serializer, huge_query ) ) ```

chunk your data

```python from queryset_iterator import queryset_iterator sync_to_other_db( session, map( serializer, queryset_iterator(huge_query) ) ) ```

imap vs. map

```python from queryset_iterator import queryset_iterator sync_to_other_db( session, map( serializer, queryset_iterator(huge_query) ) ) ```

this still realizes the entire query in memory

```python from queryset_iterator import queryset_iterator import itertools sync_to_other_db( session, itertools.imap( serializer, queryset_iterator(huge_query) ) ) ```

steps to prevent debugging

  1. you have a linter, right?
  2. access to all errors and tracebacks?
  3. flake8 should pass for tests to pass
  4. draw a line in the sand: enforce code coverage
  5. add tests to prevent regressions
